# 属性隐藏模式下的 Enter 子块创建修复计划

## 目标

修复开启 `Hide Logseq-style property lines` 后的两个编辑器结构问题：

1. 无序列表正文按 `Enter` 时，隐藏属性行、普通软换行文本和已有子树不能再被迁移。
2. 使用 Obsidian 的折叠/展开列表能力时，隐藏属性行不能再把真实子列表一起“折没”。

最终结果必须同时满足：

1. 被隐藏的属性行、普通软换行文本、fenced block 和已有子树都不能被迁移到新块下。
2. 在非空无序列表项的正文区域按普通 `Enter` 时：
   - 如果当前块已有直接子列表，新块创建为第一个子块。
   - 如果当前块没有直接子列表，新块创建为同级块。
3. 如果光标在正文中间，正文左半段保留在父块，右半段变成新子块正文。
4. 新安装用户的属性隐藏默认值重新设为开启，已有用户保持各自已保存的开关值。

## 根因结论

问题根因不是索引、块渲染或缓存，而是 Live Preview 编辑事务的语义和我们此前的插入目标不一致。

属性隐藏只是显示层装饰，真实文档里的 `key:: value`、普通 continuation 和子列表仍然存在。Obsidian 原生 Enter 按物理文档位置拆分列表项，不知道这些隐藏行在用户语义上仍属于父块。

此前已经验证过两条失败路径：

1. `keymap` 抢占 Enter：在真实 Obsidian 下没有稳定接管原生命令。
2. “统一在完整逻辑尾部后创建同级 sibling”：虽然阻止了属性迁移，但在已有子列表的块里方向仍然错了，因为用户真正要的是优先创建子块，而不是永远创建同级块。

因此最终修复必须：

1. 保留已经证明有效的 `ownerDocument` capture-phase `keydown` 拦截入口。
2. 按当前块是否已有直接子列表，动态决定是“创建子块”还是“创建同级块”。
3. 同时补一层折叠状态修复，把起始于隐藏属性行的错误折叠自动展开。

## 最终实现规则

### 命中条件

只有同时满足以下条件才由插件接管 Enter：

1. `Hide Logseq-style property lines` 已开启。
2. 纯 `Enter`，没有 `Shift/Ctrl/Alt/Meta`。
3. 非 IME 组合输入。
4. 当前 editor 只有一个空光标，没有多选区。
5. 光标位于一个非空无序列表项的正文区域内。
6. 空列表项继续走 Obsidian 原生行为。

### 新行为

命中后完全阻止 Obsidian 原生拆分，由插件一次事务完成：

1. 当前行正文从光标处分裂。
2. 左侧正文保留在父块。
3. 右侧正文变成新子块正文。
4. 父块已有的 continuation、隐藏属性、fenced block 和已有子树仍保留在父块下。
5. 如果当前块已有直接子列表，新块插在“父块自己的尾巴之后、现有第一个直接子块之前”。
6. 如果当前块没有直接子列表，新块插在“父块自己的尾巴之后、下一个 sibling/ancestor 边界之前”。

### 插入点定义

当前列表项的“父块尾巴”只包含仍属于父块自身的内容：

1. 隐藏属性行
2. 普通软换行内容
3. continuation 下的 fenced code block
4. 与上述内容相连的空行

当前列表项的“现有孩子”从第一个比父项更深的无序列表行开始。

因此新子块的插入点规则固定为：

1. 有现有孩子：插在第一个孩子之前，成为新的第一个子块。
2. 没有现有孩子：插在父块尾巴之后、下一个 sibling/ancestor 边界之前，成为同级块。

### 子块前缀规则

1. 如果当前父块已经有孩子，直接复用第一个孩子的列表前缀格式。
2. 如果当前父块还没有孩子，但需要创建子块，就按当前编辑器缩进单位生成一个更深一级的无序列表前缀。
3. 如果当前块没有孩子并创建同级块，直接复用当前块自己的列表前缀。
4. 不额外引入任务框逻辑、命令或新设置项。

### 默认值策略

`hideLogseqProperties` 改回默认 `true`，但只影响新安装或没有保存过该值的用户。已有用户继续保留自己的持久化设置。

## 实施结果要求

### 必须成功的场景

1. 父块行尾 Enter，当前块已有直接子列表：

```md
- 1A
  id:: ...
	- 2
		- 3
- 2A
```

结果：

```md
- 1A
  id:: ...
	- 
	- 2
		- 3
- 2A
```

2. 叶子块行尾 Enter，当前块没有直接子列表：

```md
- 1A
  id:: ...
	- 2
		- 3
- 2A
```

在 `3` 行尾回车后：

```md
- 1A
  id:: ...
	- 2
		- 3
		-
- 2A
```

3. 中间拆分正文，当前块没有直接子列表：

```md
- 1A
  id:: ...
	- 2B
		- 3
```

在 `2|B` 处回车后：

```md
- 1A
  id:: ...
	- 2
	- B
		- 3
```

4. continuation + 属性 + 子树混合场景中，所有尾巴都必须留在父块。

5. 折叠/展开列表后，如果子列表原本挂在一个被隐藏的属性行后面，它仍然必须重新出现，不能继续被折叠状态吞掉。

### 必须保持不变的行为

1. 空列表项 Enter 仍走原生 Obsidian。
2. 属性隐藏关闭时 Enter 仍走原生 Obsidian。
3. 多光标、非空选择、IME、修饰键 Enter 仍走原生 Obsidian。
4. Undo/Redo 必须一键回退，不允许出现“先错再补”的两阶段行为。
5. 已修好的属性行不闪现、source badge 不漂移，不能回退。

## 风险判断

这次方案比之前更稳，因为：

1. 不再依赖 Obsidian 是否把 Enter 交给自定义 keymap。
2. 不再在错误方向上修“同级插入”。
3. 不做异步搬运文本，而是一次事务原子完成。

只要 capture-phase 拦截入口继续稳定生效，再配合对隐藏属性行折叠状态的主动修复，这就是当前最直接、最完整、也最不容易回到旧 bug 的修法。
