# 复制块嵌入与返回源块计划

## 目的

这份文档记录“复制当前块嵌入命令”和“从块引用、块嵌入返回源块”两项需求的最终方案，供后续新的聊天窗口或新的工程代理直接续接。

## 需求背景

当前插件已经有：

1. `Copy current block reference`
2. UUID 风格块引用与块嵌入渲染
3. 源块旁的引用计数徽标与引用弹窗

当前还缺：

1. 一个直接复制块嵌入语法的命令
2. 从渲染后的块引用、块嵌入跳回源块的位置入口

## 已确认的产品决策

以下决策视为本轮固定方案：

1. 新增命令名：`Copy current block embed`
2. 块嵌入复制语法固定为：

```md
{{embed ((uuid))}}
```

3. `Copy current block reference` 和 `Copy current block embed` 都必须复用已有 `id:: uuid`
4. 只要当前块已经有 `id:: uuid`，就绝不重新生成
5. 命令仍然只面向无序列表源块，不放宽到任意 Markdown 行
6. 返回源块入口使用文字按钮 `Back`
7. 不使用图标，不使用 emoji
8. `Back` 按钮默认隐藏，仅在悬浮、聚焦或宿主被点击后显示
9. `Back` 不做成常驻 UI，不挤占正文阅读空间
10. 点击 `Back` 时再查询索引，不在 hover 阶段查询
11. 如果同一个 UUID 有多个活动源块位置，不自动猜测，点击后让用户选择
12. 如果源块缺失，只提示，不自动恢复，不自动改写文件

## 命令设计

### `Copy current block reference`

行为保持原有目的不变：

```md
((uuid))
```

但这一轮需要补强“已有 UUID 绝不重生”的稳定性。

### `Copy current block embed`

新命令行为：

```md
{{embed ((uuid))}}
```

### 命令共用规则

1. 光标必须位于一个无序列表块源行上
2. 如果当前行不是 `- ` 开头的大纲块，则不生成 UUID，只提示用户
3. 先查当前文件当前行是否已经被索引识别为源块
4. 如果索引还没更新，再直接扫描编辑器文本，查当前块下方是否已经存在 `id:: uuid`
5. 只有在“索引没找到 + 编辑器文本也没找到”的情况下，才生成新 UUID
6. 生成新 UUID 后，仍然按当前插件既有规则在该块软换行下插入 `id:: uuid`

### 为什么不能只依赖索引

如果只依赖索引，在这些情况里会误生成新 UUID：

1. 用户刚手工写入了 `id:: uuid`
2. 当前编辑器内容尚未被索引增量处理
3. 缓存刚加载，增量同步还没跑完

因此，命令执行时必须把“当前编辑器文本”作为第一稳定判断来源之一。

## 返回源块设计

### 适用对象

返回源块能力只作用于：

1. 渲染后的行内块引用
2. 渲染后的块嵌入

不作用于源块自身的引用计数徽标。

### 交互方式

1. 鼠标悬浮到渲染后的块引用、块嵌入上
2. 或者键盘聚焦到它们上
3. 或者先点击对应引用、嵌入宿主，把 `Back` 保持显示
4. 右上角显示一个小的 `Back` 按钮
5. 点击 `Back` 后跳回源块

### 为什么使用 `Back` 文字

1. 比图标更稳定
2. 不依赖不同系统字体或图标视觉差异
3. 维护成本更低
4. 表意直接

### 为什么不做成常驻按钮

1. 常驻会破坏阅读观感
2. 大量引用时页面噪音会明显变多
3. 这个动作属于低频操作，适合按需显示

## 跳转行为

### 单一活动源块

如果 UUID 只有一个活动源块：

1. 打开对应 Markdown 文件
2. 激活对应叶子
3. 光标定位到源块 `startLine`
4. 滚动到可见位置

### 多个活动源块

如果同一个 UUID 在多个文件里都作为活动源块存在：

1. 点击 `Back` 后不直接跳
2. 使用 Obsidian `Menu` 列出多个候选源块
3. 每条候选显示：
   - 文件路径
   - 行号
4. 用户点击其中一项后再跳转

这样设计的原因是：

1. 备份页或同步副本里可能存在同 UUID 源块
2. 自动猜测哪个源块才是“正确的”不可靠
3. 让用户选择最稳妥

### 源块缺失

如果 UUID 当前没有活动源块：

1. 提示 `Source block is missing.`
2. 不尝试恢复
3. 不改写当前位置

## 实现方案

### 命令层

在主插件类中：

1. 抽出“获取或创建当前块 UUID”的共用逻辑
2. 抽出“复制块语法到剪贴板”的共用逻辑
3. 新增 `Copy current block embed`

新增的关键辅助逻辑：

1. 计算缩进列宽
2. 扫描当前编辑器里当前列表块下方的 `id:: uuid`
3. 复用已有 UUID 而不是重生

### 渲染层

#### Live Preview

1. 行内引用的 widget 需要知道自身对应的源 UUID
2. 行内引用 widget 渲染时附带 `Back` 按钮
3. 块嵌入渲染 HTML 时附带 `Back` 按钮
4. `Back` 按钮自身带上 `data-block-ref-source-id`

#### Reading Mode

1. 行内引用元素在生成时附带 `Back` 按钮
2. 块嵌入容器在最终渲染完成后附带 `Back` 按钮
3. 错误状态、缺失状态、stale 状态下也允许显示 `Back`

### 事件层

1. 使用一个全局委托 `mousedown` 监听
2. 命中 `.block-reference-back-button` 时：
   - 读取 `data-block-ref-source-id`
   - 阻止默认行为
   - 阻止事件继续传播
   - 执行返回源块逻辑

不为每个引用或嵌入单独绑定事件。

### 索引层

索引服务新增只读查询：

1. 根据 UUID 返回当前所有活动源块位置
2. 返回值按文件路径和行号排序

这样既能支撑单源跳转，也能支撑多源选择菜单。

## 性能规则

1. `Back` 的显示只依赖 CSS hover/focus，不触发索引查询
2. 只有点击时才读取活动源块列表
3. 只增加一个全局委托监听，不给每个节点绑监听
4. 不重写现有索引主结构
5. 不改变现有反向引用徽标和引用弹窗逻辑
6. 不在 hover 阶段打开弹窗、读取文件或重新渲染

## 验证清单

### 命令验证

1. 无 ID 的大纲块执行 `Copy current block reference`，会插入 `id:: uuid` 并复制 `((uuid))`
2. 无 ID 的大纲块执行 `Copy current block embed`，会插入 `id:: uuid` 并复制 `{{embed ((uuid))}}`
3. 已有 `id:: uuid` 的块执行以上两个命令时，必须复用原 UUID
4. 即使索引尚未更新，只要编辑器文本中已经存在 `id:: uuid`，也必须复用原 UUID
5. 非无序列表行执行命令时，不生成 ID

### 返回源块验证

1. Live Preview 的行内引用能显示 `Back`
2. Live Preview 的块嵌入能显示 `Back`
3. Reading Mode 的行内引用能显示 `Back`
4. Reading Mode 的块嵌入能显示 `Back`
5. 单一活动源块时点击 `Back` 直接跳转
6. 多活动源块时点击 `Back` 弹出选择菜单
7. 选择菜单项后跳转到对应文件与行号
8. 源块缺失时点击 `Back` 只提示，不改文件

### 工程验证

1. `npm run build` 通过
2. `main.js` 正常更新
3. 大页面中悬浮与点击没有新增明显卡顿

## 非目标

本轮不做以下内容：

1. 不把 `Back` 做成图标版和文字版切换
2. 不做块引用位置的完整工具栏
3. 不做从引用位置直接恢复源块
4. 不做源块冲突自动修复
5. 不放宽源块识别规则到任意 Markdown 行
