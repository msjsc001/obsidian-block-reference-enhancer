# 源块引用计数徽标计划

## 概要

这份文档记录源块右侧引用计数徽标的最终设计，目标是让后续维护者能够直接理解当前方案。

## 目标

对“有活跃引用的源块”显示一个数字徽标。

这个徽标属于源块本身，不属于引用位置本身。

## 已确认的产品决策

1. 同时支持 Live Preview 与 Reading Mode。
2. 第一版徽标挂在源块行附近，不做最右侧独立列。
3. 点击徽标打开引用列表弹层。
4. 徽标数字表示引用出现次数，不是去重后的页面数。
5. 大量引用结果必须分页。
6. 如果同一个 UUID 在多个文件里都作为源块存在，则每个源位置都显示同样的计数。

## 数据来源

功能依赖插件已有的反向引用索引。

核心数据：

- UUID 对应的规范源块
- 当前文件中的源块位置
- 同一 UUID 的多个源块位置
- 反向引用位置列表

计数来自 `getReferenceCount(id)`。

## 预期行为

当某个源块有一个或多个活跃引用时：

- 显示数字徽标
- 点击后打开弹层
- 弹层列出引用文件、行号、引用类型和行预览
- 点击某一项可跳转到对应位置

当引用数为零时：

- 不显示徽标

## Live Preview 设计

Live Preview 由专用 CodeMirror 插件处理。

实现思路：

1. 从编辑器视图读取当前文件路径。
2. 从索引服务读取该文件内的源块。
3. 只处理可视区域附近的行。
4. 对有引用的源块行尾加 widget 装饰。
5. 在视口变化、文档变化、索引变化时刷新。

## Reading Mode 设计

Reading Mode 复用 Markdown 后处理链路。

实现思路：

1. 通过 `getSectionInfo()` 拿到当前渲染区块的行范围。
2. 通过 `MetadataCache.listItems` 确定列表项起始行。
3. 将区块内 `li` 元素与列表项缓存按顺序对应。
4. 根据源块 `startLine` 找到需要挂徽标的列表项。
5. 将同样的徽标元素插入源块内容宿主附近。

## 弹层设计

弹层同时服务于 Live Preview 与 Reading Mode。

行为要求：

- 点击徽标打开
- 再点同一个徽标关闭
- 点外部关闭
- 按 `Escape` 关闭
- 弹层内部滚动不关闭
- 点滚动条不关闭
- 页面滚动时优先重定位，而不是立刻关闭

内容结构：

- 头部先显示源块摘要
- UUID 使用缩略显示，完整值通过悬浮可见
- 每条结果显示文件名、行号、引用类型、预览文本、完整路径

## 跳转行为

点击结果项后：

1. 打开目标 Markdown 文件
2. 激活对应叶子
3. 光标跳到目标行列
4. 编辑器滚动到可见位置

## 更新规则

以下场景后必须更新徽标数字和内容：

- 文件创建
- 文件修改
- 文件删除
- 文件重命名
- 手动重建索引
- 恢复失效块

## 性能规则

1. Live Preview 只处理可视范围附近源块。
2. Reading Mode 不预先加载全部预览内容。
3. 引用结果必须分页。
4. 预览文本可按文件路径和 mtime 做缓存。
5. 没有引用的源块不制造额外 DOM 噪音。

## 验证清单

1. 验证 Live Preview 徽标显示。
2. 验证 Reading Mode 徽标显示。
3. 验证点击打开弹层。
4. 验证分页。
5. 验证点击跳转。
6. 验证增删改重命名后的计数更新。
7. 验证手动重建后的刷新。
8. 验证大页面可用性。
9. 验证同 UUID 多源位置都显示徽标。
10. 验证长路径在弹层内显示正常。
