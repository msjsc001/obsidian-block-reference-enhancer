# Support / 获取帮助

This repository uses GitHub Issues for bug reports and feature requests.

## Before opening an issue

1. Update to the latest released version of the plugin.
2. Restart Obsidian once.
3. Run `Rebuild block reference index` if the problem may be related to indexing or external file edits.
4. Check whether the issue still happens with only this plugin enabled.
5. Prepare a small Markdown sample that reproduces the problem.

## What to include in a bug report

Please include:
- plugin version
- Obsidian version
- operating system
- whether the issue happens in Live Preview, Reading Mode, or both
- exact reproduction steps
- expected result
- actual result
- console errors if available
- a minimal Markdown sample

Please redact private note content before posting.

## Where to ask for what

- Bug report: open the `Bug report` issue form.
- Feature request: open the `Feature request` issue form.
- Usage or troubleshooting: read the README and this file first, then open an issue if you still have a reproducible problem.

## 中文说明

这个仓库使用 GitHub Issues 处理 Bug 反馈和功能建议。

### 提交 issue 之前

1. 先更新到插件最新正式版。
2. 先重启一次 Obsidian。
3. 如果问题可能和索引、外部改文件有关，先执行一次 `Rebuild block reference index`。
4. 尽量确认在只开启这个插件时，问题是否仍能复现。
5. 准备一份最小可复现的 Markdown 样本。

### Bug 反馈建议附带的信息

请尽量提供：
- 插件版本
- Obsidian 版本
- 操作系统
- 问题发生在 Live Preview、Reading Mode，还是两者都有
- 明确复现步骤
- 预期结果
- 实际结果
- 控制台报错（如果有）
- 最小 Markdown 复现样本

提交前请自行脱敏，不要直接贴私人笔记内容。

### 不同问题该走哪个入口

- Bug：使用 `Bug report` 模板
- 新需求：使用 `Feature request` 模板
- 使用问题或排查问题：先看 README 和本文件，如果仍然能稳定复现，再提交 issue
